# Card Auto - Outlook 이메일 발송 가이드

카드별 엑셀을 각 사용자 이메일로 발송하는 기능입니다. Microsoft Graph API(Device Code Flow)를 사용하며, **발신자는 .env에 설정한 EMAIL_SENDER(인사팀장님 계정)** 입니다.

---

## 1. 전체 프로세스 요약

```
[1] Azure Portal 앱 등록 (최초 1회)
    ↓
[2] .env에 EMAIL_SENDER, AZURE_CLIENT_ID, AZURE_TENANT_ID 설정
    ↓
[3] 카드 사용자 마스터에 이메일 등록
    ↓
[4] 결과 파일 생성 화면 → "전체 메일 발송" 또는 개별 "발송" 버튼 클릭
    ↓
[5] 최초 1회: Outlook 인증 (브라우저에서 코드 입력 + 로그인)
    ↓
[6] 이후: 인증 토큰 자동 사용 → 자동 발송
```

---

## 2. Azure Portal 앱 등록 (최초 1회)

### 2.1 앱 등록

1. [portal.azure.com](https://portal.azure.com) 에 회사 계정으로 로그인
2. 상단 검색창에 **앱 등록** 검색 → 클릭
3. **"+ 새 등록"** 클릭
4. 아래와 같이 입력 후 **"등록"** 클릭

| 항목 | 값 |
|------|-----|
| 이름 | `CardAuto` (임의 지정 가능) |
| 지원되는 계정 유형 | 이 조직 디렉터리의 계정만 (단일 테넌트) |
| 리디렉션 URI | 비워두기 |

5. 등록 완료 후 **개요** 화면에서 아래 두 값을 복사해 보관

```
애플리케이션(클라이언트) ID  →  AZURE_CLIENT_ID
디렉터리(테넌트) ID          →  AZURE_TENANT_ID
```

### 2.2 API 권한 추가

1. 왼쪽 메뉴 **"API 사용 권한"** 클릭
2. **"+ 권한 추가"** → **"Microsoft Graph"** → **"위임된 권한"**
3. 검색창에 `Mail.Send` 입력 → 체크
4. **"권한 추가"** 클릭

> 결과: `User.Read`, `Mail.Send` 두 항목이 목록에 표시되면 정상

### 2.3 리디렉션 URI 추가

1. 왼쪽 메뉴 **"인증"** 클릭
2. **"+ 플랫폼 추가"** → **"모바일 및 데스크톱 애플리케이션"** 선택
3. 아래 URI 체크 후 **"구성"** 클릭

```
https://login.microsoftonline.com/common/oauth2/nativeclient
```

### 2.4 공용 클라이언트 흐름 허용 (필수)

1. 왼쪽 메뉴 **"인증"** 클릭
2. 상단 **"설정"** 탭 클릭
3. **"고급 설정"** 섹션에서 **"공용 클라이언트 흐름 허용"** → **"예"** 선택
4. **"저장"** 클릭

> 이 설정이 없으면 `AADSTS7000218` 오류 발생

---

## 3. 환경변수 설정

프로젝트 루트의 `.env` 파일에 추가:

```env
# Outlook 메일 발송 (Microsoft Graph - Device Code Flow)
EMAIL_SENDER=발신자이메일@회사도메인.com
AZURE_CLIENT_ID=(1단계에서 복사한 클라이언트 ID)
AZURE_TENANT_ID=(1단계에서 복사한 테넌트 ID)
```

**중요**: `EMAIL_SENDER`는 반드시 **Microsoft 365 조직 계정**이어야 합니다. Gmail 등 외부 메일은 사용할 수 없습니다.

---

## 4. 카드 사용자 이메일 등록

1. **카드 사용자** 화면 (`/lookups/cards`) 접속
2. 각 사용자 등록/수정 시 **이메일** 필드 입력
3. 이메일이 없으면 해당 사용자에게는 메일 발송이 건너뜀됨

---

## 5. 메일 발송 (웹 화면)

### 5.1 발송 버튼

- **결과 파일 생성** 화면 (`/exports`)에서:
  - **"전체 메일 발송"**: 이메일이 등록된 모든 사용자에게 일괄 발송
  - **"발송"** (개별): 해당 카드 사용자에게만 발송

### 5.2 최초 인증 (1회만)

1. "전체 메일 발송" 또는 개별 "발송" 클릭
2. 인증 다이얼로그에서 **"Outlook 인증 시작"** 클릭
3. 화면에 표시된 **접속 주소**와 **코드** 확인
4. 브라우저에서 접속 주소 열기 (예: https://microsoft.com/devicelogin)
5. 코드 입력 후 **EMAIL_SENDER**로 설정한 계정으로 로그인
6. "디바이스에서 애플리케이션에 로그인하셨습니다" 메시지 확인
7. 화면이 자동으로 발송 확인 화면으로 전환됨
8. **"발송"** 클릭하여 메일 발송

### 5.3 이후 발송

- 인증 토큰이 `data/card_auto_mail_token.json`에 저장됨
- 서버 재시작 후에도 토큰 유지
- 만료 시 자동 갱신
- 별도 인증 없이 바로 발송 가능

---

## 6. 발신자/계정 변경 시

다른 계정으로 발송하려면:

1. `.env`의 `EMAIL_SENDER` 수정
2. `data/card_auto_mail_token.json` 파일 **삭제** 또는 **내용 비우기**
3. 서버 재시작
4. 웹에서 "Outlook 인증 시작" → 새 계정으로 로그인

---

## 7. 트러블슈팅

| 오류 | 원인 | 해결 |
|------|------|------|
| AADSTS7000218 | 공용 클라이언트 흐름 미허용 | Azure Portal → 인증 → 고급 설정 → "공용 클라이언트 흐름 허용" = 예 |
| Graph API 404 (ErrorInvalidUser) | EMAIL_SENDER가 Gmail 등 외부 메일 | `EMAIL_SENDER`를 Microsoft 365 조직 계정으로 변경 |
| 이메일 주소 없음 | 카드 사용자에 이메일 미등록 | 카드 사용자 마스터에서 이메일 입력 |
